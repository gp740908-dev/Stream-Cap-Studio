## 2. Problem Statement
Content creator yang memiliki platform streaming sendiri menghadapi proses manual yang repetitif: membuka browser, menunggu stream berjalan, merekam layar, menambahkan watermark secara manual di video editor, lalu mengekspor ulang. Proses ini memakan waktu 30–90 menit per video dan rentan terhadap human error (lupa rekam, watermark salah posisi, file tidak tersimpan dengan benar).

StreamCap Studio menyelesaikan ini dengan pipeline otomatis penuh: jadwalkan → rekam → watermark → simpan → notifikasi, tanpa intervensi manual.

## 4. User Personas

### Persona 1: Andi — Solo Content Creator
- **Role**: Pemilik platform streaming drama / film indie, mengelola channel YouTube sendiri
- **Pain Point**: Harus duduk menunggu stream selesai, lalu proses manual watermark di Adobe Premiere, membuang 2–3 jam per hari
- **Goal**: Set jadwal di malam hari, pagi sudah dapat notifikasi Telegram bahwa video siap diupload

### Persona 2: Rini — Manager Konten Tim Kecil
- **Role**: Mengelola 3–5 channel YouTube untuk studio konten kecil
- **Pain Point**: Koordinasi antar anggota tim untuk proses recording sering error, file hilang, watermark tidak konsisten
- **Goal**: Satu dashboard terpusat untuk semua job, watermark brand sudah tersimpan di sistem, output selalu konsisten

---

## 5. User Stories

- Sebagai creator, saya ingin menjadwalkan rekaman dari URL stream saya agar proses berjalan otomatis tanpa saya harus standby.
- Sebagai creator, saya ingin mengupload logo watermark saya dan mengatur posisi/opacity-nya agar setiap video punya branding konsisten.
- Sebagai creator, saya ingin mendapat notifikasi Telegram lengkap (thumbnail, ukuran file, durasi) agar saya tahu rekaman selesai tanpa harus cek dashboard.
- Sebagai creator, saya ingin melihat riwayat semua job rekaman beserta statusnya agar saya bisa mendeteksi error dengan cepat.
- Sebagai creator, saya ingin output video otomatis dalam format YouTube-ready (H.264, AAC, bitrate tepat) agar tidak perlu re-encode ulang sebelum upload.

## 6. Functional Requirements

### F-01: Recording Engine
- **Description**: Otomasi Chrome via Playwright + virtual display Xvfb, merekam tab browser yang membuka URL stream menggunakan FFmpeg
- **Acceptance Criteria**:
  - [ ] Bisa membuka URL kustom di Chrome headless dengan Playwright
  - [ ] FFmpeg menangkap output Xvfb virtual display
  - [ ] Mendukung resolusi 1080p dan 720p
  - [ ] Mendukung 30 FPS dan 60 FPS
  - [ ] Auto-stop setelah durasi yang ditentukan atau ketika stream berakhir
  - [ ] File output sementara disimpan di `/tmp/streamcap/` sebelum diproses

### F-02: Watermark Engine
- **Description**: Overlay watermark PNG transparan ke video menggunakan FFmpeg filter `overlay`
- **Acceptance Criteria**:
  - [ ] Upload watermark PNG (max 5MB, harus transparan)
  - [ ] 9-point position grid (top-left, top-center, top-right, middle-left, center, middle-right, bottom-left, bottom-center, bottom-right)
  - [ ] Kontrol opacity (0–100%)
  - [ ] Kontrol ukuran watermark (% dari lebar video)
  - [ ] Margin dari tepi (px)
  - [ ] Preview watermark sebelum disimpan ke settings
  - [ ] Bisa simpan multiple watermark preset

### F-03: FFmpeg Processing Pipeline
- **Description**: Re-encode video output menjadi format YouTube-recommended
- **Acceptance Criteria**:
  - [ ] Output codec: H.264 (libx264) + AAC audio
  - [ ] Bitrate video: 8 Mbps (1080p) / 5 Mbps (720p)
  - [ ] Keyframe interval: setiap 2 detik (untuk YouTube seeking)
  - [ ] Audio: 320kbps, stereo, 48kHz
  - [ ] Container: MP4
  - [ ] File diberi nama otomatis: `[title]_[date]_[resolution].mp4`

### F-04: Job Scheduler
- **Description**: Antrian dan penjadwalan job rekaman
- **Acceptance Criteria**:
  - [ ] Buat job dengan: URL, tanggal/jam mulai, durasi (menit), resolusi, watermark preset
  - [ ] Job status: `queued` → `recording` → `processing` → `done` / `failed`
  - [ ] Bisa jalankan job sekarang (immediate) atau terjadwal
  - [ ] Celery Beat untuk trigger job terjadwal
  - [ ] Retry otomatis jika gagal (max 2x retry)

### F-05: Telegram Notification
- **Description**: Kirim notifikasi ke Telegram setelah job selesai atau gagal
- **Acceptance Criteria**:
  - [ ] Notifikasi berisi: nama job, status, durasi video, ukuran file, path output
  - [ ] Attach thumbnail frame pertama video sebagai gambar
  - [ ] Notifikasi error berisi pesan error yang readable (bukan traceback mentah)
  - [ ] Bisa test notifikasi dari Settings panel

### F-06: Job History & Logs
- **Description**: Riwayat lengkap semua job dengan detail dan log
- **Acceptance Criteria**:
  - [ ] List semua job dengan filter: status, tanggal, keyword
  - [ ] Detail job: URL sumber, waktu mulai/selesai, file output, log error
  - [ ] Download file output langsung dari dashboard
  - [ ] Hapus job dan file terkait dari dashboard
  - [ ] Retention log: simpan 90 hari terakhir

### F-07: Settings Panel
- **Description**: Konfigurasi global sistem
- **Acceptance Criteria**:
  - [ ] Telegram Bot Token + Chat ID (dengan tombol test)
  - [ ] Default storage path untuk output file
  - [ ] Default watermark preset
  - [ ] Default export preset (resolusi, FPS, bitrate)
  - [ ] Konfigurasi Chrome: user agent, viewport size, delay sebelum rekam (detik)

## 7. Non-Functional Requirements

- **Performance**: Job rekaman tidak boleh mempengaruhi responsivitas dashboard (isolated Celery workers)
- **Scalability**: Mendukung 1–5 concurrent recording job di VPS dengan RAM minimal 4GB
- **Reliability**: Auto-restart Celery worker via Docker restart policy
- **Storage**: Warning di dashboard jika disk usage >80%
- **Accessibility**: Dashboard WCAG 2.1 AA compliant
- **Compatibility**: Berjalan di Ubuntu 20.04/22.04, Debian 11/12

---

## 8. Out of Scope (V1)

- Upload otomatis ke YouTube (YouTube Data API — masuk V2)
- Multi-user dengan permission berbeda
- Cloud recording (semua lokal di VPS)
- Mobile app
- Live preview rekaman yang sedang berjalan (resource intensive)
